3D Reconstruction Fire Damaged Parchment

Team 38
Aim: a web app which processes high-quality images of fire-damaged parchments to create a 2-dimensional restored version of the antique document
Target audience: History Museums, London Metropolitan Archives

Initial research with client

  • Goals.

    Reach out to a wider, non-technical user base so that the target audience can use the embedded algorithm to reconstruct damaged parchments

  • Context of use

    Arguably every museum preserves objects with scientific, historical or educational value, that due to external factors have been damaged to the degree that they cannot be brought back to the original condition. Parchments are such objects, that owing to the nature of the material and the biological transformations that occur in the presence of fire, require tedious and thorough work to be translated properly. Our project aims to enable such institutions to reconstruct these documents so that they can export a high-standard flattened replica of the parchments.

Initial research with target audience

We interviewed Richard, an archivist at the British Museum regarding his views on a potential web application for the reconstruction of parchments. Here are our results:

  • According to Richard, ‘A web app for parchment flattening would be a terrific idea, as a lot of museums posses damaged parchments and would benefit from this technology

  • Reasons for software: preservation copy; enable visual manipulation; enhance virtual experience of manuscript

  • We’ve been advised to keep the User Interface suggestive and easy-to-use

  • We’ve been given some similar web apps we could look at such as Turning the Pages at the British Library

User requirements

  • Web app which can be used to upload images and get a 3D flattened model of the original documents

  • Intuitive UI so that non-technical audience can interact with it easily

  • Solve licensing issues

Persona

Scenarios

George Bright: Ms. Bright is a Archaeology student at Anglia Ruskin University currently working on a year abroad in Benghazi, Libya. She has encountered a fire-damaged parchment which contains information about medical ancient Berber techniques. She does not have enough fundings to use physical techniques to reconstruct the parchments. Thus, she is looking for digital tools to help her with this process.

Wang Dawei: Dawei is the director of a museum in the UK and he receives fire damaged parchments every year. He spends a large portion of his budget for these collections on treatment technologies and equipment to be able to recover information from these documents. He is investing in the development of tools which could help bring down the cost of these processes.

User control and freedom User could only go back to the main page at the end of the pipeline. They may want to go back to re-upload images at any time Put a home button at every view
Error prevention Our system only accepts images of specific formats. However, the specification is not stated in the first view Add requirements for image formats in the first view

User Testing Evaluation

We interviewed 3 users. First, we gave them initial background information about our problem and solution. Next, we let them interact freely with the initial product prototype and encouraged them to give feedback on their experience.


The feedback we received is the following:

  • There is a need to navigate more freely between pages. Hence, we have decided to have a “Back” button in all relevant pages.
  • The total number of pictures uploaded would be helpful in confirming that the upload is valid. We have included this in our post-evaluation iteration.
  • Seeing the 3D model and the flattened model next to each other would offer a meaningful comparison. Based on our target user research, we have concluded this is not the case, hence rejecting this feature.
  • More information was requested on every page, showing (1) what data is presented to the user and (2) how the user can/must interact with this data, if so. We have included this feature.
